## 1. 性能优化（对应《改进方案2》§三 第 3 批）

针对敌人数量多时的帧率隐患，做了三项优化，全部集中在 `src/main.js`（当时尚未拆分）：

- **§9 空间网格加速最近敌人查询** —— `findNearestEnemy` 对有限半径查询改走每帧构建一次的
  空间网格（`ENEMY_GRID_CELL_SIZE`），只扫描半径覆盖的格子，替代对 200+ 敌人的全量线性扫描；
  无限半径查询保留全量扫描兜底。
- **§10 时间轴阶段每帧缓存** —— `getTimelinePhase()` 按 `elapsedSurvivalMs` 记忆化（一帧内
  被数百次调用现在只算一次）；`updateEnemies` 把不稳定速度倍率、传送开关提到循环外只算一次。
- **§11 特效对象池** —— 伤害数字、冲击圈、死亡粒子、闪电线段四类高频短命对象改为对象池复用
  （`releaseToPool`），tween 结束归还而非销毁，消除战斗高峰期的 GC 抖动。

## 2. 玩法功能（对应《改进方案2》§四.12 / §四.14 第 4 批）

### §四.12 质变升级
每把武器新增一个一次性「质变」升级项，在三选一池中以紫色描边 + ★质变 徽章醒目标注，
可 reroll，每种仅出现一次：

| 武器 | 质变项 | 效果 |
|------|--------|------|
| 手枪 | 回旋弹 | 子弹到最大射程后折返飞回，回程可再次命中 |
| 突破器 | 爆炸弹 | 弹丸命中时引发小范围 AoE（复用空间网格查敌） |
| 特斯拉 | 常驻电场 | 玩家周身持续电场，每 600ms 电击近身敌人 |

常量集中在 `BALANCE.weaponUpgrades`，运行时开关在 `weaponMutations`，随每局重置。

### §四.14 元进度（localStorage）
- **存储**：`loadMetaProgress` / `saveMetaProgress` 走 localStorage，全程 try/catch，
  禁用存储时降级为内存对象不崩溃。
- **发币**：结算按 `击杀×0.5 + 存活秒×0.4 + 胜利200` 计算学分，结算面板显示「获得学分 +N（累计 M）」。
- **解锁商店**：武器选择界面显示学分 + 「解锁商店」入口，可买 4 个永久起始加成
  （强化装甲 / 轻量护具 / 军械授权 / 回收信标），买断式，下一局生效。

## 3. 界面：开始界面与暂停

- **开始游戏界面（标题页）** —— 进游戏先展示标题页（标题 + 简介 + 操作提示 + 累计学分），
  点「开始游戏」进入武器选择。流程：标题页 → 选武器/解锁商店 → 战斗。
- **暂停功能** —— 战斗中按 `ESC` 或点右上角「暂停」按钮，弹出暂停菜单（继续游戏 / 返回标题）。
  暂停时冻结物理、刷怪、计时；升级三选一与结算界面下不会误触发。
- **踩坑修复**：暂停菜单最初用 `scrollFactor(0)`，在相机跟随下点击热区与绘制位置错位导致
  按钮点不动。改用与升级菜单一致的 `syncScreenOverlayPosition`（世界空间定位）后修复。

## 6. 代码结构：拆分 main.js（对应评审 §六 / P3）

将单文件上帝类物理拆分为多模块，**运行时行为零改变**，分两阶段、经 PR #2 + CI 合并。

**阶段 1 — 配置层 `src/config/`**（纯数据，无 `this` 耦合）
- `constants.js`（尺寸/网格常量）、`balance.js`（BALANCE）、`upgrades.js`（UPGRADE_DEFINITIONS）、
  `meta.js`（元进度存储 + META_PERKS）。

**阶段 2 — 领域混入 `src/scene/`**
- 166 个方法按领域拆到 10 个文件，各导出对象字面量，main.js 用
  `Object.assign(PrototypeScene.prototype, ...)` 挂回；`constructor / create / update` 保留在 main.js。

```
src/scene/
├── menus.js       (23)  开始/武器选择/商店/暂停/结算
├── enemies.js     (36)  刷怪/AI/精英/boss/最近敌搜索
├── weapons.js     (16)  武器攻击/子弹
├── combat.js      (13)  伤害/碰撞/击退/奖励
├── effects.js     (16)  特效/粒子/对象池/音频
├── progression.js (20)  经验/升级/perk/补给
├── hud.js         (12)  UI/HUD/构建面板
├── timeline.js    (16)  时间轴/设施事件/感知干扰
├── world.js       (5)   贴图/场景/玩家/碰撞体
└── systems.js     (9)   输入/移动/暂停恢复/清理
```

**安全性**：方法体逐字迁移，`this` 语义与所有调用点不变（同一 prototype）；
169 个方法全部保留、无重复、无遗漏（diff 校验）；构建与 CI 均通过。
未做多 Scene 重架构（Boot/Game/Result），属更高回归风险项，留待后续。
