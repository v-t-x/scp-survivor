## 合并与发布门禁

- 自动化门禁：最终提交必须通过完整 Node 测试、Python 像素检查、生产构建、源码格式检查和与风险相称的浏览器 smoke。
- 源码审查门禁：复核 `31d8a24` 集成范围、无输入冲刺修复、manifest/fallback/Scene 生命周期，以及最终 diff 和 Git 状态；不得混入 cutoff 后规划、橘色路线或 App Platform 实现。
- 视觉与试玩门禁：在原生像素和实际游戏上下文检查标题、武器库、HUD、设施、角色/R-17/SCP-049、覆盖层、停电、Boss、失败、胜利和 restart，并由项目所有者完成最终验收。
- 许可门禁：53 项正式素材继续按候选素材管理，核对 `LICENSE`、`ATTRIBUTION.md` 与素材登记；SCP 衍生内容、SCP-049、生成素材权利和分发方式属于 Level 2，须由项目所有者作最终决定。
- 版本一致性门禁：README、CHANGELOG、标签与 GitHub Release 必须指向同一已验证 commit。只有明确宣称属于该版本的在线构建才必须指向同一 commit；未核对的在线 Demo 继续按独立部署状态记录。
