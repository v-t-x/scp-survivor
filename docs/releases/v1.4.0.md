# v1.4.0 — UI Foundation（回溯里程碑）

本 Release 于 2026-07-18 回溯整理，记录 2026-07-10 完成的 UI Foundation 里程碑。此回溯 Release 的唯一目标提交为 `2f9c28534d6b99f0f0996f44bb236ecb235e8d4f`；待创建的 v1.4.0 标签仅指向该原始提交。未重写历史，因此该提交内的 `package.json` 仍显示 `1.3.0`，`package-lock.json` 的根项目版本仍显示 `0.1.0`。

## 资源加载与 fallback

- 新增资源 manifest，集中声明图片、spritesheet、atlas 与音频资源 key。
- 新增 fallback texture factory：只为未成功加载的纹理 key 生成程序化回退，不覆盖已存在的正式资源。
- 新增 `PreloadScene` 作为正常启动入口，按 manifest 加载资源、补齐 fallback 后再启动 `PrototypeScene`。

## UI 与音频接口

- 新增 Theme token 目录，收集既有 UI 取值，为后续表现层接入提供稳定入口。
- 新增 `AudioManager`，继续管理 Web Audio 程序化音效、首次交互恢复、暂停/恢复与销毁。
- 新增 `UIManager` Phase 1 facade，将既有 HUD/UI 调用转发给 Scene。
- 保留 `this.playSound()` 等兼容接入，避免现有玩法模块因接口迁移改变语义。

## 集成与生命周期

- 将 Preload、Theme、`AudioManager` 与 `UIManager` 接入场景组装。
- 在 shutdown、restart 与最终 destroy 路径清理并重建 manager。
- 修复 restart 后 DESTROY 监听器累积：注册前移除同名监听器，确保 SHUTDOWN 与最终 DESTROY 的清理保持幂等。

## 发布边界

- 本版本是对既有 UI Foundation 提交的回溯记录，不包含新的历史重写或本次 release-prep 之外的功能提交。
- 不包含正式 UI/美术素材、SCP-049 压力模型或 Windows 客户端。
- 程序化纹理与 Web Audio 仍作为原型能力和资源缺失时的 fallback。
