# SCP Survivor 当前游戏设计

> **权威范围**：本文只描述 `main` 当前代码已经实现的玩法、流程和系统，不批准未来功能。产品方向见 [产品愿景](./product-vision.md)，开发授权见 [开发策略](./development-strategy.md)。

## 当前核心循环

SCP Survivor 是一个 2D 俯视角、单角色、单地图的 Survivors-like 原型。单局流程为：

1. 从标题页进入武器选择，并可在解锁商店购买永久起始加成；
2. 三选一确定本局唯一武器；
3. 使用 WASD 移动、空格闪避，武器自动攻击；
4. 击杀敌人、拾取经验、升级三选一，并处理时间轴干扰与电力故障；
5. 生存至 6:00 后清场，迎战 SCP-049；
6. 击败 Boss 获胜，生命归零则失败；两种结局都会结算学分并允许重新开始。

## 胜负与流程状态

| 状态 | 当前行为 |
|---|---|
| 标题 | 显示玩法提示，可进入武器选择 |
| 武器选择 | 三把武器中选择一把，可打开永久加成商店 |
| 战斗 | 时间推进、刷怪、自动攻击、拾取和升级 |
| 升级 | 物理与刷怪暂停，处理三选一、重抽或跳过 |
| 暂停 | `ESC` 或 HUD 暂停按钮打开；可继续或返回标题 |
| Boss | 6:00 停止常规刷怪并清场，延迟后生成 SCP-049 |
| 胜利 | SCP-049 被击败，结算含胜利奖励的学分 |
| 失败 | 玩家生命归零，按生存时间与击杀结算学分 |
| 重新开始 | 胜利或失败界面调用 Scene restart，重新进入标题流程 |

## 六分钟时间轴

| 时间 | HUD 阶段 | 当前变化 |
|---|---|---|
| 0:00 | 第一阶段：职员感染 | 以感染职员为主，常规刷怪开始 |
| 1:00 | 第二阶段：爬行者渗入 | 爬行者加入并提高压力 |
| 2:00 | 第三阶段：无人机部署 | 无人机加入；开始生成感知诱饵 |
| 3:00 | 第四阶段：电力故障 | 固定触发 25 秒停电，视野变暗且爬行者权重提高 |
| 4:00 | 第五阶段：异常介入 | 刷怪间隔缩短；敌人可短距传送；生成一次 SCP-500 |
| 5:00 | 第六阶段：收容加压 | 额外刷怪；HUD 干扰、子弹偏移与屏幕震动开启 |
| 6:00 | 终局：SCP-049 | 关闭感知干扰，停止常规刷怪，清场后进入 Boss 战 |

敌人生命、伤害、速度、种类权重与刷怪间隔随时间推进。当前设施事件只有固定的电力故障，没有随机事件轮询。

## 武器与质变

开局三把武器全部可选，整局只使用所选武器。

| 武器 | 基础定位 | 当前机制 | 一次性质变 |
|---|---|---|---|
| 基金会勤务手枪 | 中远程单体 | 自动锁定、弹丸数量与穿透成长 | 回旋弹：到达射程后折返，回程可再次命中 |
| 基金会收容突破器 | 近距爆发与控场 | 5 弹丸、4 发弹匣、装填、散布、击退、减速与硬直 | 爆炸弹：弹丸命中产生范围伤害 |
| 特斯拉发射器 | 链式范围伤害 | 在多个目标之间链击，逐链伤害衰减 | 常驻电场：周期性电击玩家附近敌人 |

Boss 阶段会优先处理 Boss 可达性：突破器按完整射程锁定 Boss 并对其造成 1.5 倍伤害；特斯拉优先锁定 Boss，并把链击用于 Boss 过载攻击。

## 升级、重抽与跳过

当前共有 16 个升级定义：6 个通用、7 个武器专属常规升级、3 个武器专属质变。升级界面从当前可用池随机给出最多 3 个不同选项。

| 范围 | 升级 |
|---|---|
| 通用 | 伤害提升、攻击速度、移动速度、最大生命值、拾取范围、紧急治疗 |
| 手枪 | 额外弹丸、弹丸穿透、回旋弹（质变） |
| 突破器 | 击退强化、压制弹、扩容弹匣、爆炸弹（质变） |
| 特斯拉 | 额外链击、快速放电、常驻电场（质变） |

- 每局初始有 3 次重抽；
- 跳过升级会恢复 8 点生命，且不超过生命上限；
- 质变每局每把武器最多获取一次；
- 达到上限或不满足武器条件的升级不会进入可选池；
- 连续获得多个等级时，会依次处理所有待选升级。

## 敌人、精英与 SCP-049

### 普通敌人

| 类型 | 特点 |
|---|---|
| 感染职员 | 基础追击近战单位 |
| 异常爬行者 | 高速、低生命近战单位 |
| 安保无人机 | 保持距离并发射投射物的远程单位 |

### 精英敌人

精英从 1:30 后开始出现，当前包含：

- 防暴镇压单位：正面减伤、冲锋，侧后方承伤更高；
- 闪现潜行者：传送预警、短距闪现与冲刺；
- 复制生物体：死亡时分裂为 3 个生物体碎片。

所有非 Boss 敌人（包括生物体碎片和 Boss 召唤物）每 6–9 秒尝试复制自身，后代继续获得独立复制 deadline；
Boss 召唤物标记会传播给后代。全局活动敌人共用 230 个硬上限，达到上限后本次复制或召唤安全截断。

### Boss：SCP-049

SCP-049 在 6:00 清场后约 2.8 秒生成，拥有 2500 生命、85 移速和 20 接触伤害。普通状态首次在登场约 5 秒后召唤，随后约每 11 秒固定请求 10 个弱化感染职员；这些单位使用 0.6 倍生命和 0.85 倍接触伤害。

Boss 使用 `normal → frenzy → normal → dying` 状态机。首次狂乱约在登场 12 秒后触发，持续 2.5 秒，之后基础冷却为 14 秒；生命不高于 50% 时，后续狂乱冷却乘以 0.65。进入狂乱会在 190px 环上固定请求 20 个全强度混合单位，类型为防暴镇压单位、闪现潜行者、复制生物体或安保无人机。狂乱期间 Boss 停止移动并暴露弱点，承伤乘以 1.35；收容突破器的 1.5 倍 Boss 增伤与其相乘后统一限制在 2.0 倍以内。状态 deadline 使用 `elapsedSurvivalMs`，暂停时不会推进。击败后进入 dying 并只触发一次胜利结算。

## 拾取物与异常物品

| 拾取物 | 来源 | 效果 |
|---|---|---|
| 经验宝石 | 敌人死亡 | 进入拾取半径后自动吸附，推动等级成长 |
| 战斗兴奋剂 | 精英敌人有概率掉落 | 恢复 15 生命，并在 6 秒内提供 20% 移速加成 |
| SCP-500 | 4:00 在地图中生成一次 | 恢复 60 生命，拾取后消耗 |

## 元进度与解锁商店

单局结束时按击杀数和生存时间获得学分，胜利额外奖励 200 学分。标题流程中的解锁商店提供 4 个一次购买、永久生效的起始加成：

| 加成 | 费用 | 效果 |
|---|---:|---|
| 强化装甲 | 150 | 起始最大生命值 +20 |
| 轻量护具 | 150 | 起始移动速度 +5% |
| 军械授权 | 250 | 起始武器伤害 +8% |
| 回收信标 | 120 | 起始拾取范围 +15% |

元进度使用 localStorage key `scp-survivor-meta`，数据结构为 `credits` 与 `perks`。读取、解析或写入失败时会回退到内存默认值，游戏继续运行。

## UI、暂停与输入

| 输入 | 功能 |
|---|---|
| `WASD` | 移动 |
| `Space` | 向当前输入方向闪避；无方向输入时沿最后朝向闪避，并获得短暂无敌 |
| `Tab` | 按住查看当前构筑面板 |
| `Esc` | 暂停或继续 |
| `M` | 静音或恢复声音 |
| `B` | 仅在 Debug 模式下直接跳至 Boss 阶段 |

当前 `DEBUG_MODE=true`，因此 `B` 跳转在现阶段构建中可用。HUD 显示生命、时间、击杀、等级、经验、当前武器状态、时间轴阶段和事件横幅。暂停界面可以继续游戏或返回标题；返回标题和结算后的重新开始都通过 Scene restart 重建本局状态。

## 当前资源与音频架构

- `PreloadScene` 是默认启动 Scene，负责读取资源 manifest、加载正式资源并为缺失 key 生成程序化 fallback；完成后启动 `PrototypeScene`；
- 当前 manifest 的图片、spritesheet、atlas 与音频清单为空，因此可玩表现仍主要来自程序化纹理与 Web Audio 合成；
- fallback factory 只为不存在的 texture key 生成纹理，不覆盖已经成功加载的真实资源；
- `AudioManager` 管理程序化音效、首次交互恢复、暂停/恢复和销毁；现有玩法仍通过 `this.playSound()` 兼容入口调用；
- `UIManager` 当前是 Phase 1 facade，把现有 HUD/UI 调用转发给 Scene；
- `PrototypeScene` 在 shutdown、restart 或最终 destroy 时清理 AudioManager 与 UIManager，并在下一次 create 时重新建立。

## 当前代码结构

- `src/main.js`：组装 `PrototypeScene`、生命周期与 manager；
- `src/scenes/PreloadScene.js`：资源加载与启动流程；
- `src/config/`：常量、平衡、升级和元进度；
- `src/scene/`：world、enemies、weapons、combat、progression、timeline、effects、hud、menus、systems 等 Gameplay mixin，以及可测试的 Boss 与复制纯规则；
- `test/`：Boss 压力合同和递归复制的 Node 回归测试；
- `src/assets/`：资源 manifest、key 与 fallback texture factory；
- `src/audio/AudioManager.js`：程序化音频；
- `src/ui/`：UI facade 与主题 token。

## 当前版本明确不包含

- 收容节点修复、区域供电操作或撤离电梯；
- 稳定化终端、协议奖励或随机设施事件系统；
- 多地图、多角色、剧情任务或完整探索结构；
- 联机、排行榜、账号、云存档或跨设备同步；
- 正式图片、spritesheet、atlas、背景音乐或外部音效资源；
- 完整中英文 i18n；当前游戏 UI 主要为中文。

这些内容是否进入未来版本由 [产品愿景](./product-vision.md)、批准后的具体计划与项目所有者决定，不能根据旧 Roadmap 自动实施。
