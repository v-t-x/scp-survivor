# UI、美术、音频与资源方向

> **权威范围**：视觉目标、混合资源策略、资源优先级与外部素材准入。具体工作仍需经过分支计划批准。

## 目标体验

- 建立统一、清晰的 SCP 基金会视觉风格；
- 提升标题、武器选择、HUD、升级、暂停和结算界面的质量；
- 提高玩家、敌人、Boss 与异常物品的识别度；
- 强化设施、警报、停电与异常事件的压迫氛围；
- 为更多 SCP、地图和角色建立可扩展资源入口。

## 当前混合资源的价值与限制

v1.6 已在玩家、R-17、SCP-049、设施、菜单、武器、升级、终端和战斗反馈中接入正式像素素材；音效仍主要由 Web Audio API 合成。程序化纹理继续保障素材缺失时可运行，并承担动态几何效果、快速调试和原型验证。

程序化表现不是需要彻底删除的遗留代码，而是正式素材缺失、开发调试与动态效果的重要 fallback。

## 混合资源原则

真实素材由 `PreloadScene` 和资源 manifest 加载；缺失 key 继续由 fallback factory 生成。正式素材优先替代高识别度、叙事性和重复出现的内容，动态几何效果继续程序化生成。

UI/Art 改造必须维持纹理 key、玩法接口、旧存档和胜负逻辑兼容，除非另有批准的接口迁移计划。

## 正式素材优先范围

- 玩家、主要敌人与 Boss；
- 地图地面与设施环境；
- 武器、升级和异常物品图标；
- 菜单、HUD 与卡片界面；
- 核心枪声、爆炸、警报与 Boss 音效；
- 背景音乐和环境音。

## 程序化生成保留范围

- 子弹轨迹与攻击范围；
- 命中、闪电、火花和粒子；
- 屏幕闪烁与异常干扰；
- Debug 图形；
- 素材缺失时的开发 fallback。

## 第一轮升级顺序

1. 核心视觉风格与 Theme；
2. HUD 与菜单；
3. 玩家与主要敌人；
4. SCP-049 Boss；
5. 地面与设施装饰；
6. 武器、升级与异常物品图标；
7. 核心音效；
8. 动画、VFX 与进一步润色。

## 玩法边界

UI、素材和音频改造不得修改伤害、生命、刷怪、AI、升级概率、任务条件或胜负逻辑。缺少玩法数据时，应提出明确接口需求，由 Project Lead 决定执行分支。

## 外部素材登记要求

外部素材进入生产资源清单前必须记录：

- 素材名称与类型；
- 作者或来源；
- 原始 URL；
- 明确许可证；
- 是否被修改及修改内容；
- 商业使用状态；
- 所需署名与传播条件。

来源不明、禁止商业使用、仅供个人使用或许可证无法确认的素材不得进入生产资源。详细门禁见 [许可与商业化准备](./licensing-and-commercialization.md)。
