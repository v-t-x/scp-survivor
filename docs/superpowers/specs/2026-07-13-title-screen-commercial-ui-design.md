# 标题页商业化战术 UI 设计

## 1. 状态与目标

- 状态：用户已确认。
- 适用分支：`feature/ui-art-overhaul`。
- 目标：在保留现有 960×540 设施门背景的前提下，把标题页从“半透明调试启动面板”升级为具有商业游戏完成度的战术标题页。
- 核心体验：玩家第一眼先识别游戏名称与失控设施，随后在一秒内找到唯一主行动入口；状态、控制提示和累计学分保持可读但退居次级。

本设计只调整标题页的构图、字体层级、控件造型和显示动效。进入军械库、学分读取、静音、存档、restart、玩法数值与胜负逻辑均保持不变。

## 2. 已选方向

采用“商业化战术标题页”：以游戏标题和收容门构成双焦点，用紧凑主行动区和底部状态栏建立清晰层级，同时保留基金会设施的专业压迫感。

未采用的方向：

- 沉浸式安保终端：世界观信息密度高，但会延续当前工具界面的观感。
- 电影式极简：氛围突出，但难以同时承载控制提示、状态与元进度信息。

本轮不新增背景、Logo 或第三方图片素材，也不添加没有真实功能的“设置、档案、退出”等菜单项。

## 3. 960×540 构图

### 3.1 背景与遮罩

- 继续使用 `title-facility-backdrop`，全屏显示，不裁换构图。
- 删除当前覆盖左侧 440×540 的均匀黑色矩形。
- 改用左深右浅的渐隐遮罩：左侧保证文字对比度，在画面中部以前自然退场，使门、管线和监控设备保持可见。
- 保留全画面轻微暗角，禁止用大面积纯黑块切断背景。

### 3.2 左上品牌标题组

标题组位于左侧安全区，距左约 52px、距上约 46px，宽度不超过 390px：

1. 顶部技术标识：`SITE-CN-03 // CONTAINMENT INCIDENT`，使用小号等宽字体。
2. 红色事故标签：`收容失效`，使用切角底板或上下细线，不再作为普通红色文本悬空。
3. 主标题分层：较小的 `SCP` 与更大的 `幸存者` 形成独立标题组合，避免现有单行巨大白字。
4. 标题附近允许使用细线、编号、短刻度和局部红色切角，但不得仿制或引入未登记的第三方 Logo。

### 3.3 任务简报

- 放在主标题下方，保持一条主要说明：`进入失控设施，完成 SCP-049 再收容。`
- 可以增加 `MISSION 01` 等短标签，但不能加入新剧情或改变目标语义。
- 操作说明从该区域移除，避免与标题争抢注意力。

### 3.4 主行动区

- 位于左下但高于底部状态栏，宽约 300–320px、高约 58–64px。
- 主文案：`01 / 进入设施`。
- 次级文案：`前往军械库并授权装备`。
- 右端保留状态灯或短箭头，按钮内部使用两级明暗、切角边框和局部高光，禁止恢复为圆角网页按钮或空荡的大灰框。
- 页面保持唯一主行动入口；激活后仍调用现有 `beginFromStartScreen()`。

### 3.5 右侧门禁焦点

- 不新增独立图片；用 Phaser Graphics 在现有收容门周围绘制低占用的定位角标、细线和 `GATE 03` 标记。
- 标记围绕门体布置，不跨过门的主要轮廓，不形成新的大型面板。
- 门上方警报区域提供低亮红色呼吸光和短扫描标记，使背景成为叙事的一部分。

### 3.6 底部状态栏

- 在底部约 44–48px 高度建立单条横向状态栏，使用低透明深色底和顶部分隔线。
- 左侧：`WASD 移动 / SPACE 闪避 / TAB 构建 / ESC 暂停 / M 静音`。
- 中部：电力在线与收容失效状态，使用现有 contained/danger 色和状态灯。
- 右侧：`累计学分 0000`，读取现有 `this.meta.credits`。
- 三段信息必须对齐，不再以多行散落文本占据主视觉区域。

## 4. 动效与交互

### 4.1 入场

总时长约 600ms：

1. 背景从约 102% 平滑回落到 100%，不改变最终裁切。
2. 标题组先从左侧 12–18px 淡入。
3. 任务简报与主行动区随后错峰淡入。
4. 右侧门禁标记最后点亮。

动效用于建立层级，不得阻塞输入；玩家在入场期间点击主按钮仍应正常进入军械库。

### 4.2 常驻氛围

- 警报灯低频呼吸，完整周期约 1.4–2.0 秒。
- 门禁定位线每数秒完成一次短距离扫描。
- `收容失效` 标签仅做轻微亮度脉冲。
- 禁止持续屏幕抖动、高频闪烁、大面积扫描线和影响文字阅读的异常抖动。

### 4.3 按钮状态

- Idle：清楚的默认边框、两级内部明暗和可见状态灯。
- Hover：边框与局部高光增强，主文字向右移动约 3px，状态灯变亮。
- Pressed：短暂压暗，保持现有点击反馈。
- Activated：可短暂显示 `授权中`，但不得延迟或改变现有场景切换语义。

键鼠热区必须与视觉按钮完整重合；不能用不可见的额外对象覆盖右侧门禁或底部状态栏。

## 5. 组件边界与数据流

- `src/scene/menus.js`：负责标题页业务组装、现有文本数据和进入军械库回调。
- `src/art/titleBackdrop.js`：负责背景遮罩、门禁标记、警报氛围和标题页专属 tween 生命周期。
- `src/ui/tacticalUi.js`：如需支持主按钮的主/副文案或位移动效，应通过向后兼容的可选参数扩展；现有军械库与 HUD 调用不得改变。
- `src/ui/theme.js`：只增加可复用的标题页 token，禁止复制新的散落颜色体系。

数据流保持单向：`createStartScreen()` 读取 `this.meta.credits` 并创建显示对象；主按钮激活后调用 `beginFromStartScreen()`；`destroyStartScreen()` 负责停止控制器并销毁该页面拥有的全部对象。

## 6. 生命周期与异常处理

- 标题页控制器必须集中持有所有 tween，并提供幂等 `stop()` 或 `destroy()`。
- `destroyStartScreen()`、Scene shutdown 和 restart 后不得残留 tween、监听器、遮罩或门禁标记。
- 正式背景缺失时继续依赖现有 manifest/fallback 合同；UI Graphics 和文本仍应完整显示。
- 对未知或极大学分值使用右对齐和最大宽度约束，避免底部状态栏溢出。
- 不修改 `AudioManager`、`UIManager`、Scene 启动顺序、localStorage schema 或 restart 语义。

## 7. 验证

### 7.1 自动测试

- 标题页不再创建固定 440×540 的纯色左侧 scrim。
- 主标题、事故标签、任务简报、主行动区和底部三段状态均存在。
- 主行动区仍唯一调用 `beginFromStartScreen()`。
- 新增的战术按钮可选参数不改变现有调用结果。
- 标题页控制器停止后所有 tween 均停止，重复 stop/destroy 安全。
- restart 路径不增加重复监听器。

### 7.2 视觉 Smoke

在 Phaser WebGL、960×540、DPR 1 下检查：

- 标题和收容门形成清楚双焦点，背景不再被左侧黑块截断。
- 主按钮可在一秒内定位，idle、hover、pressed 可明显区分。
- 底部控制、设施状态和学分均完整且不抢主标题。
- 入场、警报和扫描动效不会造成裁切、闪烁或输入阻塞。
- 从标题页进入军械库、暂停返回标题以及失败重启后，页面均只创建一套 UI 与 tween。
- 捕获视觉证据时连续取多个 WebGL 候选帧，拒绝清帧与重绘之间产生的撕裂截图。
