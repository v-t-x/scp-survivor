# 肩载火控模块表现系统设计

## 1. 状态与目标

- 状态：用户已确认核心表现、模块家族和技术边界。
- 适用分支：`feature/ui-art-overhaul`。
- 目标：解决自动攻击方向与人物移动朝向不一致造成的视觉违和，把自动瞄准解释为基金会制式肩载火控装备，并让现有武器升级获得可见反馈。
- 核心体验：人物身体表达移动与生存动作，固定在背包承力架上的武器模块独立瞄准和开火；玩家能立即理解子弹为什么射向人物未面对的目标。
- GPT 审查等级：Level 1。该设计引入新的角色装备幻想和表现系统，但不改变伤害、冷却、目标选择、弹道、胜负或存档语义，默认无需外部 GPT 审查。

本设计只授权表现层接入、正式素材、UI 文案和必要的只读攻击事件通知。不得借机修改武器平衡、自动瞄准规则、敌人 AI、投射物碰撞、升级概率或 Scene 架构。

## 2. 背景与根因

当前人物动画朝向由移动速度决定，自动攻击角度由武器系统根据最近目标计算。玩家向左移动时可以向右上目标自动射击，但人物素材中手持武器仍沿移动方向显示，枪口火光和弹丸又从人物中心生成，因此画面会呈现“枪口朝左、子弹朝右”的矛盾。

以下方案已比较：

- 攻击时强制人物转向目标：素材改动小，但怪群中会频繁抖动转身，削弱移动可读性。
- 人物与手持武器分层：最自然，但需要完整重做手臂/武器方向层，并持续处理武器与身体遮挡。
- 漂浮武器：能解释自动攻击和额外弹丸，但不符合当前基金会设施的专业、克制风格。
- 贴身肩载火控模块：自动瞄准设定明确，不要求人物转身，且可把升级反馈整合进同一套装备。

采用最后一种方案。

## 3. 核心表现

### 3.1 人物职责

- 人物身体只表达移动、待机、闪避、受伤和死亡。
- 正式玩家 spritesheet 改为低姿态操作员：双手持控制握把、扫描器或低垂备用装备，不再出现一把明显正在朝移动方向射击的主武器。
- 保持现有 `player-opening-sheet` 纹理 key、576×192 尺寸、48×48 单帧、四方向×十二帧结构、脚底基准和碰撞体合同。
- 角色显示尺寸、physics body、移动速度和朝向算法均保持不变。

### 3.2 模块职责

- 模块固定在背包承力架上，不悬浮，不绕人物公转。
- 承力架跟随人物位置和深度；炮臂与武器头独立转向攻击目标。
- 瞄准表现量化为八个方向，避免低分辨率像素素材任意旋转产生模糊边缘。
- 实际攻击角度仍可连续变化；短枪口曳光负责连接八方向模块姿态与真实弹道。
- 无目标时模块在约 200–300ms 后回到肩后行军位置，避免待机时持续乱转。
- 暂停、结算和标题/军械库阶段不更新瞄准或后坐动画。

### 3.3 开火反馈

- 枪口火光、后坐、供弹机构和蓄能灯均从肩载模块发出。
- 实际投射物继续使用现有玩家物理位置、攻击角度、射程和碰撞规则。
- 用长度受限的短曳光或枪口连接帧掩盖模块枪口与现有弹丸起点之间的十几像素视觉偏移；不得移动 bullet body 或改变 `originX/originY`。
- Tesla 的真实连锁计算保持不变，只增加模块到第一目标方向的蓄能与起始闪光。

## 4. 三类模块家族

### 4.1 M-01 Sentinel / 勤务手枪模块

- 轻型双轴承力架，瞄准与回正速度最快。
- 现有“额外弹丸”升级改名为“并联火控”，描述改为增加一个同步射击通道。
- `projectileCount` 仍保持现有 1–5 和扇形弹道语义；表现上由同一模块逐步增加可见枪管、发射节点或火控灯，不增加整把漂浮枪。
- 穿透等其他升级不在本批新增玩法表现，除非实施计划明确列入纯视觉反馈。

### 4.2 B-07 Breach / 收容突破模块

- 重型短管、液压炮臂和明显机械复位。
- 现有弹仓容量通过可见供弹仓长度或弹壳节点表现。
- 换弹期间炮臂下压、供弹仓闪烁或开启；只读取现有 `isReloading`、`currentShells` 和 `magazineSize`。
- 击退或抑制升级可以增强后坐、液压火花和排气，但不能改变实际击退参数。

### 4.3 T-03 Arc / 特斯拉放电模块

- 枪体替换为线圈阵列和短距放电头。
- 连锁目标增加时亮起更多线圈节点；冷却缩短时只加快蓄能环节奏。
- 放电前允许短促蓄光，真实攻击仍在现有冷却到期时立即执行，表现动画不得引入攻击延迟。
- Tesla Field 等质变效果继续沿用现有玩法语义，本批只允许连接视觉状态。

## 5. 组件与数据流

### 5.1 纯表现规则

新增 `src/art/weaponRigPresentation.js`，负责：

- 将连续攻击角度映射为八方向表现姿态；
- 校验有限角度并保留最后一个有效方向；
- 将 `weaponId` 与只读升级状态映射为模块外观；
- 生成行军、瞄准、开火、换弹和蓄能状态；
- 输出冻结的展示状态，不读取或修改 Scene。

### 5.2 Phaser 视图控制器

新增 `src/art/weaponRigView.js`，创建并集中持有承力架、武器头、枪口、状态灯和短曳光对象。控制器至少提供：

- `update(anchor, presentationState, deltaMs)`：跟随人物并更新姿态；
- `fire(snapshot)`：播放一次纯视觉开火反馈；
- `setPaused(paused)`：停止或恢复表现更新；
- `destroy()`：幂等停止 tween、销毁对象并释放监听器。

控制器不得读取 Scene 的伤害、敌人、胜负或升级选择逻辑，也不得返回影响攻击成功与否的值。

### 5.3 Scene 接入

- 玩家创建完成后创建 `weaponRigController`；若资产或控制器创建失败，继续运行当前攻击表现。
- 武器在成功找到目标并确定 `baseAngle` 后，向可选控制器发送只读快照：`weaponId`、`aimAngle`、攻击时间和当前已有展示参数。
- 手枪快照可附带 `projectileCount`；突破器附带 `currentShells`、`magazineSize`、`isReloading`；Tesla 附带 `chainTargets` 和冷却比例。
- 可选通知不得参与 `didAttack`、`nextAttackAtMs`、伤害或目标选择控制流。
- 更新循环只传递人物锚点、暂停状态、设施异常强度和 delta；模块不自行搜索敌人。

数据流保持单向：

`现有武器攻击状态 → 纯展示快照 → weaponRigPresentation → weaponRigView`。

## 6. 资源、manifest 与 fallback

- 继续使用现有 `player-opening-sheet` key 替换正式玩家 PNG，避免更改角色加载接口。
- 为三类肩载模块新增明确、唯一的生产纹理 key；manifest 修改原因仅为加载已批准的正式模块素材。
- 每个模块提供八方向武器头或等价 atlas；承力架、状态灯和短曳光可部分使用 Phaser Graphics。
- fallback factory 必须为每个新模块 key 提供简化几何模块，确保正式素材缺失时可运行。
- 新增或替换的角色、模块、图标和派生素材必须更新 `docs/art/asset-register.md`，记录来源、生成方式、修改状态、商业使用复核和署名要求。
- 不引入来源不明的第三方图片、字体或音频。

## 7. 生命周期与异常处理

- Scene restart、shutdown、返回标题和最终 destroy 时，控制器必须幂等销毁全部显示对象、tween、定时器和监听器。
- 无任务、暂停、升级覆盖层和结算状态下模块保持静止，不继续产生枪口或曳光。
- 未知 `weaponId`：显示通用收起模块，不播放开火反馈。
- 非有限 `aimAngle`：保留最后一个有效姿态；若从未获得有效角度则使用行军姿态。
- 缺失纹理：使用程序化 fallback；控制器缺失时武器攻击继续执行，不抛出运行时错误。
- 停电与异常干扰只改变模块灯光、透明度或瞄准线稳定度，不改变目标和命中。
- 所有高频枪口、曳光和电弧对象复用现有 transient/pool 约定，避免每次攻击产生无法回收的对象。

## 8. 玩法与职责边界

本批允许：

- 读取已经存在的攻击角度和升级状态；
- 调整纯视觉枪口、曳光、后坐、灯光和角色正式素材；
- 修改升级名称与说明，使其匹配“并联火控”等表现，但保持 apply 逻辑不变；
- 为表现层增加可选通知入口。

本批禁止：

- 修改自动目标选择、目标优先级和攻击触发条件；
- 修改 bullet body、出生物理位置、速度、伤害、射程、穿透或弹道；
- 修改 `projectileCount`、弹丸上限、霰弹数量、连锁数量和升级概率；
- 修改 AI、刷怪、Boss、胜负、六分钟时间轴、localStorage 或 Scene 拆分。

若实现发现必须移动真实弹丸起点或改变攻击时序，停止实施并作为 `dev/v2` 的显式接口/玩法提案另行批准。

## 9. 测试与视觉验证

### 9.1 自动测试

- 连续角度到八方向姿态的边界、负角度和非有限值处理。
- 三类武器状态映射，以及手枪 1–5 个并联火控通道。
- `fire()` 只创建/更新表现对象，不写入伤害、速度、physics body、AI、冷却和胜负状态。
- 可选控制器缺失时三种武器攻击返回值与现状一致。
- 暂停时不更新；恢复后不补播过期攻击。
- `destroy()` 重复调用安全，restart 后不存在重复模块、tween 或监听器。
- 正式模块素材缺失时 fallback 完整，未知武器进入收起状态。
- 玩家替换 spritesheet 保持 48 帧、尺寸、脚底、方向体量和现有 physics geometry 合同。

## 10. 回退策略

- 肩载模块由可选控制器接入；回退时移除表现通知即可恢复当前攻击显示，武器玩法不需要迁移。
- 正式模块纹理缺失时自动使用 fallback，不阻断运行。
- 玩家正式 PNG 可按单一资产 commit 回退，纹理 key 和 animation 合同保持稳定。
- 不新增存档字段或迁移步骤，因此回退不影响现有 `localStorage` 数据。
