# v1.6 急救包泛化设计

## 1. 目标

把当前借用旧异常编号、但只提供一次性回血效果的拾取物完整泛化为普通医疗补给，中文公众名称为“急救包”，英文公众名称为 “Medkit”，内部标识统一为 `medkit`。

本轮只修正身份、命名和相关说明，不改变生成时机、生成位置、治疗数值、碰撞、提示时序、音效、存档、六分钟流程或现有程序化图形。一个只有普通回血效果的拾取物不再借用具体 SCP 编号，也符合项目“不把 SCP 只当名称与美术包装”的产品原则。

## 2. 已确认事实

- 当前物品在 4:00 生成一次，恢复 60 生命。
- 生成点位于玩家附近 240–420 距离，并夹取在距离世界边界至少 72 的范围内。
- 拾取物碰撞圆半径为 9、depth 为 14，使用 `pickupHeal` 音效。
- 拾取效果按“治疗提交 → UI 更新 → 拾取物销毁 → 650ms HUD cue”执行。
- 顶部事件横幅持续 2400ms。
- 当前图形是 18×18 的红白程序化 fallback，没有对应的正式 PNG，也没有使用旧条目图片。
- 相关生成状态只存在于单局内，没有进入 `localStorage`，因此内部标识迁移不需要存档迁移。

## 3. 命名与玩家文案

统一采用以下名称：

| 范围 | 旧名称 | 新名称 |
|---|---|---|
| 中文公众名称 | 旧异常编号 | 急救包 |
| 英文公众名称 | 旧异常编号 | Medkit |
| 内部标识 | 旧专有标识 | `medkit` |
| 时间配置 | 旧专有生成时间标识 | `medkitSpawnAtMs` |
| 单局状态 | 旧专有生成状态 | `medkitSpawned` |
| 更新方法 | 旧专有更新方法 | `updateMedkitSpawn()` |
| 生成方法 | 旧专有生成方法 | `spawnMedkit()` |
| 纹理键 | 旧专有纹理键 | `TEXTURES.medkit` / `"medkit"` |
| 拾取与 HUD reason | 旧专有 reason | `"medkit"` |

顶部横幅改为：

- 标题：`医疗补给`
- 详情：`急救包已在设施内出现`

README 英文使用 “Medkit”。中文正式文档使用“急救包”，不把玩家黑话“血包”作为正式名称。

## 4. 修改范围

### 4.1 运行时与测试

完整重命名配置、Scene 状态、更新/生成方法、纹理键、拾取类型和 HUD cue reason。现有聚焦测试先改为新契约并观察失败，再实施最小生产代码使其通过；另加一个急救包身份测试，锁定名称与全部不变量。

程序化图形的 18×18 尺寸、红白配色和绘制几何保持不变。本轮不新增、删除或重画 PNG，不调整 manifest 的资源数量。

### 4.2 当前文档与历史说明

README、CHANGELOG 和 `docs/design.md` 改为“急救包 / Medkit”，并把只描述普通补给的“异常物品/异常拾取”改为“医疗补给/补给拾取”。

为保证 v1.6 当前 tracked tree 不再继续分发旧条目身份，`docs/archive/` 与 `docs/superpowers/` 中的相关名称和示例标识也同步泛化。Git 历史、旧标签和既有 GitHub Release 不重写；已公开历史版本的许可事实不会因当前 HEAD 改名而自动改变。

### 4.3 许可边界

当前 v1.6 tree 完整移除旧编号、名称、条目特有治愈设定和视觉表达后，新的普通急救包不再需要该旧条目单项署名。`ATTRIBUTION.md` 当前没有该条目，因此不新增也不删除该项。

项目仍使用 SCP Foundation 世界观与 SCP-049，现有总体 SCP/CC BY-SA 候选说明、SCP-049 署名和 53 项生成素材的最终许可决策继续保留。本修正只关闭该旧条目的单项缺口，不替代 v1.6 的 Level 2 总体许可门禁。

## 5. 不变量与非目标

以下内容必须保持不变：

- `240000ms` 生成时机；
- 240–420 生成距离与 72 边界夹取；
- 60 生命治疗量；
- 碰撞圆半径 9、depth 14；
- `pickupHeal` 音效；
- 治疗、UI、销毁和 HUD cue 的提交顺序；
- 650ms HUD cue 与 2400ms 横幅时长；
- 18×18 程序化图形的最终外观；
- 六分钟流程、数值、敌人、AI、胜负和 `localStorage` 语义；
- 53 个生产 PNG、51 个运行时加载与 2 个历史保留素材的统计。

本轮不新增异常规则、不把急救包重新设计为有代价的 SCP 物品，也不修改 SCP-049、橘色路线、角色动画或 App Platform。

## 6. 验证与门禁

实施至少完成：

1. 新旧拾取契约的 TDD RED/GREEN 证据；
2. 急救包聚焦测试；
3. 全部 Node 测试；
4. Python 像素测试；
5. Vite 生产构建；
6. Markdown 本地链接检查；
7. tracked tree 中旧编号、旧内部标识和旧条目特有治愈词零命中；
8. manifest、素材统计、cutoff、排除提交、版本号、隐私与 `git diff --check` 门禁；
9. 独立规格符合性审查和代码质量审查；
10. 浏览器 smoke，确认横幅显示“医疗补给 / 急救包已在设施内出现”且拾取恢复 60 生命。

这是玩家可见修改，因此新 HEAD 必须重新绑定项目所有者视觉/试玩确认，旧 HEAD `6738be3` 的验收不能自动覆盖新提交。push、PR、merge、tag 和 GitHub Release 仍分别需要原计划规定的具体授权。

## 7. 完成标准

- 运行时、测试、当前文档和随 v1.6 源码归档分发的历史说明统一使用 `medkit`、急救包和 Medkit；
- 当前 tracked tree 不再包含旧条目身份；
- 所有玩法和视觉不变量得到测试或 diff 证据支持；
- 新候选通过完整自动化、独立审查、浏览器 smoke 和项目所有者复验；
- 总体许可门禁仍被准确标记为待 Level 2 与项目所有者决定，不因本次泛化被误报为全部解决。
