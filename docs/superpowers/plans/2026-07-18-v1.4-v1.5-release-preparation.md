# v1.4.0 与 v1.5.0 发布整理实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 准备可审查的 `v1.4.0` 回溯发布资料，并把当前 `main` 收口为版本元数据、运行状态和公开文档一致的 `v1.5.0` 发布候选。

**Architecture:** `v1.4.0` 只记录历史 UI Foundation 边界，不重写旧提交；`v1.5.0` 通过当前 main 之上的 release-prep 提交同步版本、发布模式和文档。发布说明保存在 `docs/releases/`，后续经确认后原样用于 GitHub Release。

**Tech Stack:** Markdown、Git、GitHub Releases、Node.js、Phaser 3、Vite 7、JavaScript ES Modules。

## Global Constraints

- `v1.4.0` 的唯一标签目标是 `2f9c28534d6b99f0f0996f44bb236ecb235e8d4f`，且必须称为回溯里程碑。
- `v1.5.0` 的功能完成边界是 `daeb3ed605f29a2d97a30458ee12109561dbaf6d`，标签只能指向本轮全部提交通过门禁后的最终 release-prep HEAD。
- 根 `package.json` 与 `package-lock.json` 的项目版本必须同时为 `1.5.0`。
- `src/config/constants.js` 的 `DEBUG_MODE` 必须为 `false`，`docs/design.md` 必须描述同一事实。
- UI/Art、App Platform 与角色动画路线不进入本轮已发布内容。
- 本轮不创建标签、不创建 GitHub Release、不 push、不 merge、不 rebase、不 squash、不重写历史、不部署 Demo。
- README、CHANGELOG 与状态文档按发布后的最终公开状态准备，但只保留在隔离发布分支；未获得标签、Release 与 main push 的再次授权前，不得把这些最终态文案推到公开 `main`。
- 不修改 `docs/archive/` 中的历史记录，不暂存 `dist/`、`node_modules/`、`.superpowers/` 或本地私有材料。
- 新增或实质修改的项目文档使用中文；`README.en.md` 只保持对应的英文事实镜像。

- [ ] **Step 1: 验证本地 Markdown 链接**

Run:

```powershell
$files = @('README.md','README.en.md','CHANGELOG.md','docs/README.md','docs/project-status.md','docs/design.md','docs/development-strategy.md','docs/releases/v1.4.0.md','docs/releases/v1.5.0.md')
$broken = foreach ($path in $files) {
  $file = Get-Item -LiteralPath $path
  $text = Get-Content -LiteralPath $path -Raw -Encoding utf8
  foreach ($match in [regex]::Matches($text, '\[[^\]]+\]\(([^)]+)\)')) {
    $target = $match.Groups[1].Value
    if ($target -match '^(https?://|mailto:|#)') { continue }
    $relative = ($target -split '#', 2)[0]
    if (-not $relative) { continue }
    $resolved = Join-Path $file.DirectoryName $relative
    if (-not (Test-Path -LiteralPath $resolved)) { "$path -> $target" }
  }
}
if ($broken) { $broken; throw 'broken local Markdown links' }
```

Expected: 无输出、退出码 0。
